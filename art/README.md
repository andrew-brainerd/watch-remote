# App icon art

The app icon is generated with Gemini (`gemini-3-pro-image`) and rendered into every platform icon
size by `pnpm icons`.

## Files

| Path | Committed | What it is |
|---|---|---|
| `icon-master.png` | yes | **The approved icon.** 1024×1024, PNG, opaque RGB. Single source of truth. |
| `icon.txt` | yes | The generation prompt. Contains a `{{PALETTE}}` placeholder. |
| `palettes/*.txt` | yes | Swappable color blocks (`amber`, `blue`, `purple`). Purple is the shipped one. |
| `icon-fixup.txt` | yes | Image-to-image prompt for touching up an existing render without redrawing it. |
| `generated/` | **no** | Raw candidates + preview contact sheets. Gitignored (~17 MB). |

## Regenerating the icon set from the master

```sh
pnpm icons
```

This runs `tauri icon` and then **strips the alpha channel from the iOS AppIcon set**. That second
step is not optional: `tauri icon` always writes those PNGs as RGBA, and App Store Connect rejects
the 1024×1024 marketing icon if it carries an alpha channel — the *upload* fails, not the build.
`src-tauri/gen/` is gitignored and regenerated by `tauri ios init`, so re-run `pnpm icons` after any
regeneration.

## Generating new art

The generator script lives in the docs repo and reads `GEMINI_API_KEY`, which is in the
**brainerd-api** Infisical project — so `infisical run` must be invoked from that directory:

```sh
cd ~/projects/brainerd-api
AR=1:1 infisical run --silent -- node ~/projects/docs/wedding/art/gen.mjs \
  gemini-3-pro-image <out.png> <prompt.txt> [reference-images...]
```

To try a palette, substitute the block into the prompt first:

```sh
python3 -c "
base=open('art/icon.txt').read(); pal=open('art/palettes/blue.txt').read().strip()
open('/tmp/icon-blue.txt','w').write(base.replace('{{PALETTE}}', pal))"
```

### Gotchas learned the hard way

- **Output is JPEG data with a `.png` name.** Gemini returns JPEG bytes; `tauri icon` refuses it
  ("Invalid PNG signature"). Convert before use: `Image.open(src).convert('RGB').save(dst,'PNG')`.
- **Never say "app icon" in the prompt.** It makes the model draw a rounded-square badge with white
  corners, which double-masks on iOS. Describe it as a *square poster that fills the canvas*.
- **Verify the corners numerically**, not by eye — sample the four corner pixels and confirm none are
  white. Over-pressuring "no rounded corners" once produced stepped pixel-art corner ornaments
  instead, so check for decoration too.
- **Check legibility at 40 px** before approving. Thin limbs, small buttons and low-contrast accents
  all dissolve; big shapes and a bright screen panel survive.
